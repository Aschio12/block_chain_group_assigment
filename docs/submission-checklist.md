# Submission Checklist

## Mandatory Items
- [ ] All 5 members have visible public commits.
- [ ] Course materials are added in `materials/`.
- [ ] Course codebases are added in `course-work/`.
- [ ] Progress/completion screenshots are added in `screenshots/`.
- [ ] Repository is clearly organized in folders.
- [ ] Final repository link is ready for submission.

## Quality Checks Before Submission
- [ ] No empty placeholder folders only.
- [ ] File names are clear and consistent.
- [ ] README is up to date.
- [ ] Screenshots are readable.
- [ ] Commit history reflects real contributions from each member.
